\column{0.47\textwidth}
\begin{block}{Research setting}
AIR1 asks for room-level occupancy over zones 1--15. The evidence is not one
desk, one camera, or one sensor stream: two camera views, per-zone masks,
shared AIR-1 readings, SEN55 readings, and readiness gates all shape the result.
\end{block}
\tightsep
\begin{itemize}
  \item Camera coverage differs by zone and timestamp.
  \item Missing camera coverage should stay unknown, not become vacancy.
  \item The dashboard must remain useful while production inference is gated.
\end{itemize}
\end{columns}
\end{frame}

\begin{frame}[shrink=4]{Research Questions}
\begin{enumerate}
  \item Can one shared temporal model represent zones 1--15 from long-form \code{timestamp + zone_id} rows?
  \item Can per-zone CV labels stay null when coverage is missing instead of becoming false negatives?
  \item Can \code{zone_id} support grouping, audits, splits, and display without entering the model input?
  \item Can the all-zones dashboard expose camera freshness and readiness before a production model is promoted?
\end{enumerate}
\tightsep
\begin{center}
\parbox{0.84\textwidth}{\centering
\small The deck expands the applied-research story into a technical review of source acquisition, labels, features, windowing, model architecture, validation, serving, deployment, and limitations.}
\end{center}
\end{frame}

\begin{frame}[shrink=4]{Label Contract}
\begin{columns}[T,onlytextwidth]
\column{0.48\textwidth}
\textbf{Target semantics}
\begin{itemize}
  \item Target: \code{occupied}.
  \item Key: \code{timestamp + zone_id}.
  \item Positive rule: median \code{occupancy_count >= 1}.
  \item Negative rule: median count is below 1.
  \item Missing label rule: \code{occupied = null}.
\end{itemize}

\column{0.48\textwidth}
\textbf{What null protects}
\begin{itemize}
  \item Camera not covering a zone at a timestamp.
  \item Missing per-zone payload evidence.
  \item Late or absent MQTT messages.
  \item Validation from treating unknown as confidently empty.
\end{itemize}
\end{columns}
\tightsep
\begin{block}{Contract test coverage}
\small Regression tests assert that missing per-zone labels stay null and that Table 16 is excluded from cleaned labels.
\end{block}

\begin{frame}[shrink=4]{Optuna And Training Loop}
\begin{columns}[T,onlytextwidth]
\column{0.50\textwidth}
\textbf{Search space}
\begin{itemize}
  \item Lookback, batch size, learning rate, weight decay.
  \item Conv block count, channels, kernel sizes.
  \item Activation, normalization, pooling, global pooling.
  \item Dense units, dropout, scheduler, patience.
\end{itemize}

\column{0.46\textwidth}
\textbf{Training mechanics}
\begin{itemize}
  \item Optimizer: AdamW.
  \item Class imbalance handled with positive weight from train labels.
  \item Schedulers: none, cosine, or reduce-on-plateau.
  \item Objective: mean validation PR-AUC across eligible folds.
  \item Final model trains on all pre-test rows.
\end{itemize}
\end{columns}
\tightsep
\begin{block}{Shared model implication}
\small AIR1 searches one model configuration for all zones rather than fitting isolated per-zone models.
\end{block}
\end{frame}

\begin{frame}[shrink=4]{Split And Coverage Policy}
\begin{columns}[T,onlytextwidth]
\column{0.50\textwidth}
\textbf{Strict path}
\begin{itemize}
  \item Latest calendar day is held out as blind test.
  \item Pre-test dates form one-day rolling validation folds.
  \item Timestamp groups keep all zones for one timestamp together.
  \item Valid lookbacks must fit train and validation windows.
\end{itemize}

\column{0.46\textwidth}
\textbf{Coverage evidence}
\begin{itemize}
  \item Per-zone label coverage is checked.
  \item Both-class windows are required for meaningful validation.
  \item Rejected lookbacks and under-covered dates are recorded.
  \item Bootstrap fallback is first-production recovery only.
\end{itemize}
\end{columns}

  \item The latest error explains blockers such as pointer absence.
\end{itemize}

\column{0.48\textwidth}
\textbf{Health does not prove}
\begin{itemize}
  \item A production model has already been promoted.
  \item Every zone has current labels.
  \item Sensor/API features are complete.
  \item A candidate model passed promotion.
\end{itemize}
\end{columns}
\tightsep
\begin{center}
\parbox{0.82\textwidth}{\centering\small The dashboard is allowed to be operationally useful before production inference is ready, as long as readiness is explicit.}
\end{center}
\end{frame}

\begin{columns}[T,onlytextwidth]
\column{0.48\textwidth}
\textbf{Useful before promotion}

  \item cam1 and cam2 MJPEG feeds show view freshness.
  \item Per-zone CV truth can be displayed independently from model probability.
  \item Health payload reports missing production pointer or source blockers.

support data collection and diagnosis before the first promotable all-zones

occupancy.

\begin{frame}[shrink=4]{Deployment And Operations}
\begin{columns}[T,onlytextwidth]
\column{0.50\textwidth}
\textbf{Persistent runtime}
\begin{itemize}
  \item User-systemd services own camera loops, aggregator, SEN55 collector, live collector, app, and trainer timer.
  \item Generated data, logs, and model runs remain package-local and untracked.
  \item The trainer lock prevents overlapping full fits.
\end{itemize}

\column{0.46\textwidth}
\textbf{Operational readiness}
\begin{itemize}
  \item Enable trainer timer only after collection and label coverage are mature enough.
  \item Check pointers, status JSON, service state, and health routes before declaring readiness.
  \item Keep Zone 5 deployment detail out of AIR1 claims.
\end{itemize}
\end{columns}
\tightsep
\begin{block}{Manual-first rule}
\small AIR1 automation should follow a successful proof run; scheduled training should not obscure whether coverage, split, or model gates are the real blocker.
\end{block}
\end{frame}

\begin{frame}[shrink=4]{Limitations And Threats To Validity}
\begin{columns}[T,onlytextwidth]
\column{0.49\textwidth}
\textbf{Data and label threats}
\begin{itemize}
  \item Camera occlusion and mask coverage vary by zone and view.
  \item Missing camera coverage creates null labels and reduces usable evidence.
  \item Table 16 exclusion limits model scope to zones 1--15.
  \item Calendar-window maturity can block strict rolling validation.
\end{itemize}

\column{0.47\textwidth}
\textbf{Operational threats}
\begin{itemize}
  \item Production pointer absence means inference is not ready even when collection is alive.
  \item Sensor/API availability affects environmental feature freshness.
  \item Training quality depends on enough occupied and unoccupied windows per split.
  \item Shared modeling may hide zone-specific failure modes without per-zone audits.
\end{itemize}
\end{columns}
\end{frame}

\begin{frame}[shrink=4]{Roadmap}
\begin{columns}[T,onlytextwidth]
\column{0.50\textwidth}
\textbf{Near-term hardening}
\begin{itemize}
  \item Mature per-zone label coverage before treating all-zones inference as production-ready.
  \item Promote only after strict-window evidence, smoke checks, and contract tests pass.
  \item Keep camera freshness, null label rates, and production pointer state visible.
\end{itemize}

\column{0.46\textwidth}
\textbf{Research extensions}
\begin{itemize}
  \item Add per-zone reliability summaries for reviewers and operators.
  \item Compare shared-model errors across camera coverage groups.
  \item Extend contract only after a source-grounded reason to include new fields.
\end{itemize}
\end{columns}
\end{frame}
